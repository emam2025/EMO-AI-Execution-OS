## 2. Bridge Points Requiring IsolationRuntime (Gaps)

### Gap A: ExecutionEngine → Node Execution (RULE 1 enforcement)
- **Current**: `ExecutionEngine.execute()` calls `self._pool.submit(self._runtime.execute_node_safe, ...)` directly.
- **Problem**: Bypasses `IsolationRuntime` for individual node execution.
- **Fix**: Node execution MUST go through `IsolationRuntime.execute()`.

### Gap B: ThreadPoolExecutor in ExecutionEngine (RULE 4)
- **Current**: `self._pool = ThreadPoolExecutor(max_workers=4)` with `self._cancel_flag = Event()`.
- **Problem**: Workers are not individually killable. Cancel flag is coarse-grained.
- **Fix**: `SandboxManager` should manage per-node worker lifecycle, not `ExecutionEngine`.

### Gap C: Non-Daemon Threads in ControlPlane
- **Current**: `control_plane.py:68,95` creates non-daemon threads.
- **Problem**: Violates RULE 4 — threads survive shutdown.
- **Fix**: Convert to daemon threads or wrap in `SandboxExecutor.execute_direct()`.

### Gap D: Direct `open()` calls in runtime components
- **Current**: Various runtime components use `open()` for config/log files.
- **Problem**: No `IOPolicyEngine` bypass for internal runtime IO.
- **Fix**: Define internal IO as always-allowed via `IOPolicyEngine.set_policy("runtime", "*", IOPolicy(allowed=True))`.

### Gap E: Import-level OS binding
- **Current**: `import subprocess`, `import socket`, `import tempfile` in sandbox modules.
- **Problem**: Module-level imports create unavoidable OS coupling.
- **Fix**: Keep in sandbox modules — this is intentional (sandbox IS the OS boundary).

## 4. Bridge Architecture (Target)

```
ExecutionEngine
  └── IsolationRuntime (BRIDGE)          ← NEW: single entry point
        ├── CapabilityGuard.validate()    ← Step 1
        ├── ResourceEnforcer.check()      ← Step 2
        ├── SandboxManager.create()       ← Step 3
        │     └── SandboxExecutor         ← OS boundary (subprocess)
        ├── SandboxExecutor.execute()     ← Step 4
        └── ResourceEnforcer.finish()     ← Step 5
```

**Key rule**: `ExecutionEngine` MUST NOT call `self._pool.submit()` directly.  
All node execution MUST route through `IsolationRuntime.execute(tool_name, inputs)`.
